# Отчет по лабораторной работе №7: Анализ генерации поисковых запросов ИИ-моделями

## Цель работы

Проанализировать качество генерации поисковых запросов для научной литературы двумя различными ИИ-моделями (ChatGPT 5 и Claude 3.5 Sonnet) на основе научной статьи о стохастическом многокритериальном принятии решений.

## Критерии оценки результатов

1. **Достоверность и проверяемость источников**
2. **Релевантность теме**
3. **Покрытие подтем (полнота)**
4. **Разнообразие без дублей**
5. **Воспроизводимость и прозрачность**

## Предложенные промпты

### Промпт 1: Простые поисковые запросы

```
Сгенерируй 12 простых поисковых запросов для сайта arXiv (https://arxiv.org/search), которые точно дают выдачу. Требования:
- Английский язык, 2–4 общих ключевых слова без кавычек, без двоеточий и логических операторов.
- Не используй редкие аббревиатуры и длинные составные фразы.
- В каждом запросе должен быть хотя бы один «якорный» термин: learning, neural, model, algorithm, method, system, analysis, dataset — выбери подходящие к теме.
- Выведи только готовые строки запросов, по одному на строку.
```

### Промпт 2: Пакеты запросов

```
Подготовь 10 пакетов запросов для сайта arXiv (https://arxiv.org/search). Каждый пакет — два web‑safe варианта, которые дают выдачу:
- Basic: 3–6 ключевых слов без кавычек и операторов (только пробелы).
- Soft‑phrase: одна короткая фраза в кавычках до 3 слов + 1–2 общих слова (например: "graph neural" models).
Ограничения:
- Только английский; избегай редких узких терминов и длинных фраз.
- Не используй NOT/AND/OR, поля (ti:, abs:, cat:) и двоеточия.
- В каждом варианте добавляй 1 «якорное» слово (learning, neural, model, algorithm, method, system, analysis, dataset).
Формат вывода: для каждого пакета две строки подряд — сначала Basic, затем Soft‑phrase.
```

### Промпт 3: Эскалируемые запросы

```
Сформируй 10 наборов эскалируемых запросов для arXiv по теме. Для каждого набора дай три строки:
- Q0 (web‑safe): 3–5 общих ключевых слов без кавычек и операторов — ориентирован на https://arxiv.org/search (All fields).
- Q1 (web‑safe+): одна короткая фраза в кавычках (до 3 слов) + 1–2 общих слова. Без операторов.
- Q2 (fielded мягкий, для API или Advanced Search): abs:"<краткая фраза до 3 слов>" OR ti:"<краткая фраза до 3 слов>" — без NOT; категорию не добавляй.
Правила:
- Только английский; избегай редких аббревиатур (кроме общеизвестных вроде GNN, LLM, NLP).
- Не используй длинные точные фразы.
- Не показывай рассуждения; выведи только тройки строк в порядке Q0, Q1, Q2 для каждого набора.
```

### Промпт 4: Фасеты и типы запросов

```
Выдели 6 фасетов (подтем) по тексту ниже (каждый 2–3 слова), НО не выводи их отдельно. Для каждого фасета сгенерируй по 3 запроса, предназначенные для arXiv и дающие выдачу:
- Minimal (web‑safe): 2–4 общих ключевых слова, без кавычек и операторов.
- Broadened (web‑safe): добавь 1–2 синонима через пробел (без OR), оставь якорные слова (learning, neural, model, method, system).
- Review‑oriented (web‑safe): включи слова survey или review вместе с темой, без кавычек и операторов.
Ограничения:
- Только английский. Запрещены NOT/AND/OR, поля (ti:, abs:, cat:), двоеточия и длинные точные фразы.
- Избегай узких редких терминов; предпочитай общеупотребимые.
Формат вывода:
- 18 строк (по 3 на каждый из 6 фасетов) в группах по 3 строки подряд: Minimal, затем Broadened, затем Review‑oriented.
```

## Собственный промпт

```
Generate 15 high-quality search queries for arXiv (https://arxiv.org/search) based on the scientific article about stochastic multiple criteria decision making. Each query should be optimized for finding relevant academic papers.

Requirements:
- All queries in English, web-safe format (no special operators except quotes for exact phrases)
- Focus on core concepts: stochastic multiple criteria decision making (SMCDM), dominance degrees, probability distributions, PROMETHEE II method, ranking alternatives
- Include relevant anchor terms: method, algorithm, model, system, analysis, dataset
- Provide 3 types of queries for each of 5 thematic clusters:

Thematic clusters:
1. Core SMCDM methods and algorithms
2. Stochastic dominance and probability approaches
3. PROMETHEE II and outranking methods
4. Decision analysis and ranking systems
5. Applications and case studies

For each cluster, generate:
- Precise query: 2-4 specific keywords with 1 anchor term
- Balanced query: short phrase in quotes (2-3 words) + 1-2 related keywords
- Exploratory query: broader terms to discover related work

Format: List queries by cluster, clearly marking the type of each query.
Ensure diversity and avoid duplicates across all queries.
```
